1. Access the application using port-forward:

  export POD_NAME=$(kubectl get pods --namespace {{ .Values.namespace.name }} -l "app.kubernetes.io/name={{ include "speedtest.name" . }},app.kubernetes.io/instance={{ .Release.Name }}" -o jsonpath="{.items[0].metadata.name}")
  echo "Visit http://127.0.0.1:{{ .Values.deployment.container.port }} to use your application"
  kubectl --namespace {{ .Values.namespace.name }} port-forward $POD_NAME {{ .Values.deployment.container.port }}:{{ .Values.deployment.container.port }}

  Alternative one-liner:
  kubectl port-forward -n {{ .Values.namespace.name }} deployment/{{ .Values.deployment.name }} {{ .Values.deployment.container.port }}:{{ .Values.deployment.container.port }}

2. Speed Test Application has been deployed!

   - Namespace: {{ .Values.namespace.name }}
   - Application: {{ .Values.deployment.name }}
   - Replicas: {{ .Values.deployment.replicas }}
   - Image: {{ .Values.image.repository }}:{{ .Values.image.tag }}

3. Configuration:
   - Small File URL: {{ .Values.env.smallFileUrl }}
   - Medium File URL: {{ .Values.env.mediumFileUrl }}
   - Large File URL: {{ .Values.env.largeFileUrl }}
   {{- if eq (include "speedtest.useImagePullSecrets" .) "true" }}
   - Image Pull Secret: {{ include "speedtest.imagePullSecretName" . }}
   {{- else }}
   - Image Pull Secret: Not configured (using public image)
   {{- end }}

4. Check the deployment status:
   kubectl get pods -n {{ .Values.namespace.name }} -l app={{ .Values.labels.app }}

5. View logs:
   kubectl logs -n {{ .Values.namespace.name }} -l app={{ .Values.labels.app }} --follow

Thank you for using Speed Test Application!
